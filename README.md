## 📁 Struktura Projektu

Poniżej znajduje się opis zawartości poszczególnych katalogów w tym repozytorium:

### 📄 [01_test_cases](./01_test_cases)
* **Opis:** Pliki zawierające napisane przeze mnie przypadki testowe (Test Cases).
* **Zakres:** Testy funkcjonalne, testy formularzy logowania/rejestracji, koszyka zakupowego itp.
* **Format:** [np. Pliki Excel / PDF / Link do Jira].

### 🐛 [02_bug_reports](./02_bug_reports)
* **Opis:** Przykładowe raporty o błędach (Bug Reports) wykrytych podczas testów.
* **Zawartość:** Każdy raport zawiera jasny tytuł, kroki do odtworzenia (Steps to Reproduce), wynik oczekiwany vs. faktyczny, priorytet (Severity/Priority) oraz załączone screenshoty/logi.

###  [03_checklists](./03_checklists)
* **Opis:** Listy kontrolne (Checklisty) używane do szybkiego zweryfikowania poprawności działania kluczowych modułów aplikacji przed wydaniem.
* **Zastosowanie:** Testy regresji oraz testy dymne (Smoke Tests).
